# Shield-Vote MVP Implementation Plan

## Overview
Build a complete, runnable MVP of "Shield-Vote" — a blockchain-based e-voting system with Edge AI liveness/coercion detection, deployed on a local Hardhat network with a React/Vite frontend.

## Architecture

```
 MetaMask / Demo Wallet
        │
 React + Vite Frontend ──── Hardhat Local Node (localhost:8545)
        │                          │
   ┌────┴────┐              ShieldVote.sol
   │         │               (castVote, getResults, verifyVote)
 MediaPipe  SHA-256
 Edge AI    Commitment       Mock IPFS
 (WASM)     (Web Crypto)     (fake CIDs)
```

## User Flow
Home → Auth (voterID + secret → SHA-256 commitment) → Liveness (AI challenges) → VotePage (select candidate, cast via Ethers.js) → Confirmation (TX hash + CID) → VerifyVote (enter hash → verify on-chain)

## Phase 4: Edge AI Components

### `frontend/src/components/LivenessCheck.jsx`
- Init FaceLandmarker from @mediapipe/tasks-vision (VIDEO mode, outputFaceBlendshapes: true)
- Webcam via getUserMedia
- Challenge state machine (shuffled: blink, turn_left, turn_right, smile)
- **Blink**: Eye Aspect Ratio (EAR) < 0.2 from eye landmarks
- **Head turn**: Nose tip x vs face midpoint; ±15% displacement
- **Smile**: Blendshapes mouthSmileLeft + mouthSmileRight > 0.5, or mouth width/height ratio
- Security: no face → alert, multiple faces → block
- Props: `onPass` callback

### `frontend/src/components/CoercionMonitor.jsx`
- Background component on VotePage
- FaceLandmarker + PoseLandmarker running simultaneously
- Suspicion scoring system:
  - Multiple faces → immediate trigger
  - Rapid blinking (>3/sec) → +2
  - Gaze away >3s → +1
  - Repeated head turns → +2
  - Abnormal posture → +1
  - No face >2s → +2
  - Score decays -1 per 5s; trigger at score >= 5
- On coercion: cast dummy vote (candidateId=0), show alert, offer override button
- Props: commitmentHash, onCoercionDetected

## Key Technical Decisions

1. **MediaPipe**: Use `@mediapipe/tasks-vision` (modern unified package), load WASM from CDN
2. **IPFS**: Mocked — no real Pinata API needed
3. **Wallet**: MetaMask with JsonRpcProvider demo fallback
4. **Ethers.js**: v6 exclusively (BrowserProvider, async getSigner, native BigInt)
5. **State**: React Context (no Redux), useState/useEffect
6. **Vite config**: MUST set `optimizeDeps.exclude: ['@mediapipe/tasks-vision']` and COOP/COEP headers

---

## Known Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| MediaPipe WASM needs SharedArrayBuffer headers | Set COOP/COEP in vite.config.js |
| Vite pre-bundles WASM incorrectly | Exclude @mediapipe/tasks-vision from optimizeDeps |
| Webcam stream shared between Liveness and Coercion | Stop tracks in useEffect cleanup; consider shared CameraContext |
| Coercion false positives | Suspicion score accumulation (threshold 5) with decay |
| Contract address changes on redeploy | Update DEPLOYED_ADDRESS in contract.js after each deploy |
| Ethers v5 vs v6 API confusion | Use v6 exclusively: BrowserProvider, async getSigner |
